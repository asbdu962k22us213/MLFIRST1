# Student Exam Score Predictor

A beginner-friendly Machine Learning web application that predicts a student's final exam score from study hours, attendance percentage, sleep hours, and previous exam marks.

The project uses Flask for the backend, HTML/CSS/JavaScript for the frontend, pandas and NumPy for dataset handling, and scikit-learn Linear Regression for prediction.

## Features

- Home page with a responsive prediction form
- Synthetic dataset generation with around 500 rows
- Linear Regression model training
- Model saved with pickle
- Flask API route: `POST /predict`
- JSON response from the backend
- Frontend uses Fetch API to call Flask
- JavaScript form validation
- Loading animation while predicting
- Result card with animated score bar
- Error handling on frontend and backend

## How the Model Works

`train_model.py` creates a synthetic dataset with these input features:

- `study_hours`
- `attendance`
- `sleep_hours`
- `previous_marks`

The target column is:

- `exam_score`

The script trains a `LinearRegression` model from scikit-learn. After training, it saves:

- `dataset/student_data.csv`
- `model/student_model.pkl`

The Flask app loads `student_model.pkl` and uses it for predictions.

## How Frontend and Backend Communicate

1. The user fills out the form in `templates/index.html`.
2. `static/js/script.js` validates the values.
3. JavaScript sends the values to Flask using `fetch("/predict")`.
4. Flask receives JSON data in `app.py`.
5. Flask predicts the score using the saved model.
6. Flask returns JSON like:

```json
{
  "prediction": 82.45
}
```

7. JavaScript displays the result dynamically on the page.

## Installation Steps

Open the project folder in VS Code, then open the terminal.

### 1. Create a Virtual Environment

Windows:

```bash
python -m venv .venv
```

### 2. Activate the Virtual Environment

PowerShell:

```bash
.\.venv\Scripts\Activate.ps1
```

Command Prompt:

```bash
.venv\Scripts\activate
```

If PowerShell blocks activation, run:

```bash
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

Then activate again.

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Train the Model

```bash
python train_model.py
```

This creates the dataset CSV and model pickle file.

### 5. Start the Flask Server

```bash
python app.py
```

Open this URL in your browser:

```text
http://127.0.0.1:5000
```

## File Explanation

### `app.py`

Creates the Flask application. It renders the home page at `/`, accepts prediction requests at `POST /predict`, validates input, loads the saved model, predicts the score, and returns JSON.

### `train_model.py`

Generates a synthetic dataset, trains a Linear Regression model, evaluates it using MAE and R2 score, saves the CSV file, and saves the trained model with pickle.

### `templates/index.html`

Contains the HTML structure for the web page, including the hero section, input form, message area, and prediction result card.

### `static/css/style.css`

Styles the frontend with a gradient background, responsive layout, cards, hover effects, animations, and mobile-friendly design.

### `static/js/script.js`

Handles form validation, loading state, Fetch API requests to Flask, success messages, error messages, and dynamic result display.

### `requirements.txt`

Lists all Python packages needed to run the project.

### `dataset/student_data.csv`

Generated by `train_model.py`. Contains the synthetic student dataset.

### `model/student_model.pkl`

Generated by `train_model.py`. Contains the trained Linear Regression model used by Flask.

## Common Beginner Mistakes and Fixes

- **Problem:** `ModuleNotFoundError: No module named 'flask'`
  **Fix:** Activate your virtual environment and run `pip install -r requirements.txt`.

- **Problem:** `Model file not found`
  **Fix:** Run `python train_model.py` before running `python app.py`.

- **Problem:** PowerShell does not activate the virtual environment
  **Fix:** Run `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`, then activate again.

- **Problem:** Browser shows an old design
  **Fix:** Hard refresh the page with `Ctrl + F5`.

- **Problem:** Port 5000 is already in use
  **Fix:** Stop the other Flask app or run this app on another port by changing `app.run(debug=True)` to `app.run(debug=True, port=5001)`.
